# Design

The full design source is `celis-design-system.md`. Read it before UI work.

## Rules

- Preserve established Celis tokens, brand language, layout rules, and motion patterns.
- Use existing Radix/Tailwind UI primitives.
- Use reusable admin tables, status badges, confirm dialogs, pagination, and form sections.
- Listing UI should prioritize clarity, trust, image quality, pricing, location, and seller context.
- Admin UI should prioritize density, scanning, filtering, and safe actions.

## Do Not

- Do not introduce a second design system.
- Do not create one-off action buttons when shared components exist.
- Do not perform destructive or financial actions without confirmation UI.
